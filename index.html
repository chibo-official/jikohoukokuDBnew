取込状況
取込ルール
  • 伝票No. をユニークキーとして重複チェック — 既存IDはスキップ
  • 申請日 のExcelシリアル値を YYYY/MM に自動変換して月次集計に使用
  • 伝票状態は取込対象外（無視）
  • 取込後、月別集計シートを自動再構築
  • 取込履歴はGoogleスプレッドシートの「取込ログ」シートに記録
セットアップ手順
初回のみ必要な設定です（所要時間：約20分）
GAS エンドポイント URL 設定
詳細手順
  1. 1
    Google スプレッドシートを新規作成
    Google スプレッドシート を開いて新しいファイルを作成。 URLから スプレッドシートID をコピー:
    https://docs.google.com/spreadsheets/d/【ここがID】/edit
  2. 2
    Google Apps Script を設定
    スプレッドシートメニュー → 拡張機能 → Apps Script を開く。
    Code.gs の内容を貼り付け。
    先頭の SPREADSHEET_ID に手順1のIDをセット。
    保存（Ctrl+S / ⌘+S）。
  3. 3
    GAS をウェブアプリとしてデプロイ
    右上「デプロイ」→「新しいデプロイ」→ 種類を「ウェブアプリ」。
    実行するユーザー: 自分（アカウント名）
    アクセスできるユーザー: 全員
    「デプロイ」ボタン → 表示される URL をコピー。
  4. 4
    このページに URL を設定
    上のテキストボックスにデプロイで取得したURLを貼り付け → 保存して接続テスト。
    接続成功が表示されれば完了。URLは localStorage に保存されます。
  5. 5
    GitHub Pages に公開（任意・社内共有向け）
    GitHubに accident-report-db リポジトリを作成。
    index.html をコミット & プッシュ。
    リポジトリ Settings → Pages → Source を main ブランチに設定。
    https://【ユーザー名】.github.io/accident-report-db/ で公開完了。
  6. 6
    毎月の更新作業
    ① 千房の事故報告書Excelを準備
    ② このページ「データ取込」メニューからファイルをアップロード
    ③ 伝票No.で重複チェックし、新規データのみ自動追加
    ④ ダッシュボードが自動更新される

    ※ Excelファイルの列順: 伝票No. / 申請日 / 申請者コード / 申請者名 / 入力者コード / 入力者名 / 所属部門コード / 所属部門名 / 事故内容 / 発生日時 / ... / 損害額 / 確定損害額 / 負担対象
